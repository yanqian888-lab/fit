# 减肥搭子 APP —— 产品需求文档（PRD）

## 1. 产品概述

### 1.1 产品定位

「减肥搭子」是一款基于 3 个 AI Agent 协作的聊天式减肥陪伴 APP。核心定位是 **"你的专属 AI 减肥搭子"**，让用户像和真实朋友聊天一样完成减肥记录、获得情绪支持和专业建议。

### 1.2 核心价值主张

| 价值点 | 说明 |
|--------|------|
| 聊天即记录 | 用户无需手动填写，聊天内容自动沉淀为饮食、运动、体重等记录 |
| 有温度的陪伴 | 搭子不是冰冷教练，而是会偷懒、会嘴馋、懂情绪的减肥伙伴 |
| 个人减肥博物馆 | 自动沉淀金句、感悟、食谱、对比照，形成专属减肥纪念 |
| 极致低成本 | MVP 零成本启动，1000 DAU 以内月成本控制在 50 元以内 |

### 1.3 目标用户

- 18-35 岁有减肥需求的年轻女性（主力）和男性
- 尝试过传统减肥 APP 但觉得记录繁琐、缺乏陪伴的用户
- 希望有人监督、鼓励、一起坚持的减肥人群

## 3. 核心功能详细说明

### 3.1 聊天搭子体验

#### 3.1.1 进入方式

打开 APP 默认进入聊天界面，所有功能围绕聊天展开。

#### 3.1.2 搭子人设

- 名字：默认「瘦瘦」，用户可自定义
- 身份：和用户一起减肥的真实普通人
- 特点：会偷懒、会嘴馋、会平台期、会偷吃后愧疚
- 永远站在用户这边，理解情绪，不批评、不说教

#### 3.1.3 三种模式

| 模式 | 语气 | 口头禅 | 处理典型场景 |
|------|------|--------|--------------|
| 温柔鼓励型 | 温柔耐心像闺蜜 | "没关系的"、"已经很棒了" | 用户难过时先安慰再鼓励 |
| 严格监督型 | 坚定直接不啰嗦 | "不行"、"忍住"、"坚持住" | 用户偷懒时直接指出并给方法 |
| 损友互怼型 | 幽默毒舌互调侃 | "笑死"、"你可拉倒吧" | 用玩笑方式监督，不让减肥压抑 |

#### 3.1.4 搭子实时状态

- 显示状态：起床、吃饭、上班、上学、运动、睡觉
- 每 2 小时自动更新，模拟真实生活节奏
- 搭子身体状态与用户基本同步

### 3.2 自动沉淀系统

#### 3.2.1 沉淀内容分类

| 一级分类 | 二级分类 | 同步目标 |
|----------|----------|----------|
| 核心业务类 | 饮食记录、运动记录、身体数据、生活习惯 | 记录中心 |
| 个人资产类 | 减脂食谱、减肥方法、踩坑教训、好物工具 | 博物馆 |
| 情感成长类 | 励志金句、心路感悟、情绪记录 | 博物馆 |
| 里程碑类 | 体重里程碑、时长里程碑、行为里程碑 | 时间轴 + 成就 |
| 多媒体类 | 身材对比照、食物照片、运动照片 | 记录中心 / 博物馆 |
| 计划目标类 | 每日计划、每周计划、欺骗餐计划 | 提醒系统 |
| 问题咨询类 | 常见问题、专业建议 | 帮助中心 / 方法库 |

#### 3.2.2 置信度处理策略

| 置信度区间 | 处理策略 | 用户感知 |
|------------|----------|----------|
| ≥ 0.95 | 自动确认，直接沉淀 | 消息下方显示绿色 "已记录" |
| 0.80 - 0.94 | 自动沉淀，显示提示 | 消息下方显示 "已记录" 提示 |
| 0.70 - 0.79 | 标记为待确认 | 显示待确认卡片，用户可一键确认 |
| < 0.70 | 不沉淀 | 仅聊天回复，不记录 |

#### 3.2.3 沉淀流程

```
用户发送消息
    ↓
主协调 Agent 立即回复用户（< 2 秒）
    ↓
异步转发给信息沉淀 Agent
    ↓
信息沉淀 Agent 提取信息 → 计算置信度
    ↓
按置信度策略处理 → 同步到记录中心 / 博物馆
    ↓
通知主协调 Agent 在聊天界面显示状态标记
```

### 3.3 记录中心

#### 3.3.1 今日概览

- 热量缺口大卡片：摄入 / 消耗 / 剩余
- 颜色状态：绿色健康 / 黄色预警 / 红色超标
- 今日体重显示
- 本周体重变化趋势

#### 3.3.2 饮食记录

- 按早餐、午餐、晚餐、加餐分类
- 自动提取的饮食显示 "待确认" 标记
- 支持一键确认、编辑、删除
- 内置常见食物数据库（MVP 阶段先用精简版）
- 支持手动添加食物

#### 3.3.3 运动记录

- 按有氧运动、力量运动分类
- 自动提取的运动显示 "待确认" 标记
- 支持一键确认、编辑、删除
- 内置常见运动消耗数据

#### 3.3.4 身体数据

- 每日体重记录与差值计算
- 最近 7 天体重变化折线图
- 目标体重设置，显示距离目标还有多少斤
- 支持编辑和删除历史记录

### 3.4 我的减肥博物馆

#### 3.4.1 博物馆总览

- 核心进度总卡片：已减重量、已用时间、目标完成率
- 预计完成目标剩余天数
- 可视化进度条
- 减肥总天数统计
- 成就徽章墙（MVP 阶段展示基础徽章）

#### 3.4.2 时间轴历程

- 按时间倒序展示重要减肥事件
- 事件类型：体重变化、里程碑、金句、感悟、照片
- 支持点击事件查看详情、编辑、删除

#### 3.4.3 分类内容库

- 金句墙：区分 "我的金句" 和 "搭子的金句"
- 食谱库：自动收集减脂食谱，支持一键添加到今日饮食
- 感悟集：自动收集心路历程
- 方法库：支持标记 "亲测有效" / "无效"

## 4. 三 Agent 协作机制

### 4.1 Agent 角色定义

| Agent | 角色 | 使用模型 | 核心职责 |
|-------|------|----------|----------|
| 主协调 Agent | 大脑 + 感性层 | doubao-seed-2.0-lite | 对话上下文管理、意图识别、情绪识别、共情话术生成、最终回复 |
| 信息沉淀 Agent | 独立后台 | doubao-lite-32k | 聊天记录实时扫描、7 大类信息提取、置信度计算、数据同步 |
| 全能助手 Agent | 理性层 | doubao-seed-2.0-lite | 身体指标计算、营养评估、运动方案、专业问题解答 |

### 4.2 调用流程

```
用户发送消息
    ↓
主协调 Agent 接收消息（响应时间 < 2 秒）
    ├─ 立即生成初步回复返回给用户
    ├─ 异步转发完整消息给信息沉淀 Agent
    └─ 判断是否需要专业计算
        ↓ 是
    调用全能助手 Agent → 补充回复用户
        ↓
信息沉淀 Agent 处理消息
    ├─ 提取信息 → 计算置信度
    ├─ 置信度 ≥ 0.95 → 自动同步数据库 → 通知主协调显示 "已沉淀"
    ├─ 0.7 ≤ 置信度 < 0.95 → 生成待确认卡片 → 主协调显示给用户
    └─ 置信度 < 0.7 → 丢弃
```

### 4.3 交互协议

- 主协调 → 全能助手：函数 `call_allround_helper(question, user_data)`
- 主协调 → 信息沉淀：函数 `call_precipitation(content, user_id)`
- 工具调用格式：`<<<FunctionCall>>>[{"name":"函数名","parameters":{}}]<<<FunctionCallEnd>>>`

## 5. 非功能性需求

### 5.1 性能要求

- 聊天首次回复时间 < 2 秒
- 页面切换加载时间 < 1 秒
- 历史消息加载支持分页

### 5.2 成本控制

- 所有计算优先在本地完成（BMI、BMR、热量缺口等）
- 所有 AI 调用优先使用豆包免费模型
- 服务器使用最低配置（2 核 2G）
- 所有数据默认本地存储，云端备份可选
- 不使用任何付费第三方服务（推送、短信、统计等）

### 5.3 安全与隐私

- API Key 等敏感信息存放服务器环境变量
- 用户数据加密存储
- 支持账号注销与数据永久删除
- 聊天记录和沉淀数据仅用户本人可见
